　　1.3 Description of device permission invocation
 1) Storage permission (read/write storage) : if the android version is Android 10 or lower, App will apply Storage permission  to store the necessary application configuration files and task test data files, signaling  export, test track export, etc., your storage permission needs to be invoked.
 2) Album permissions: When you try to import the indoor map, you need album permissions.
 3) Call rights: Call functions are required when you need to test call tasks. this task including Outgoing call or Incoming call
 4) Location permission: outdoor map page displays real-time track and current network information, service cell identification, signal strength display
 5）Access telephone equipment: if you want to lock rat and lock frequency,app need apply this privilege.
 6)  After obtaining your permission, the app will attempt to use root privileges, mainly for frequency locking, ratlocking, obtaining signaling and more wireless parameters. The  prerequisite for execution is that your device has already been rooted and the chipiset is Qualcomm and MTK.

 

 [Remark]
（1）Due to the need to render the outdoor dot track in real time, the frequency of GPS acquisition is 1-2 seconds.
（2）The frequency of the base station is collected in 1-2 seconds as it needs to render the trend diagram of cell level in real time.
（3）Android needs to access GPS to obtain cell information. Disabling GPS will result in failure to display network signal strength.。
（4）Disabling the permission to access telephone equipment information will cause the locking network frequency locking function can not work properly.
